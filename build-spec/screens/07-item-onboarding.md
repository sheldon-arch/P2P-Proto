# Screens: 07 Item and Catalog Onboarding (Show-light)

Flow 07 realized as screens. Source: `documentation/07-item-onboarding.md`, data dictionary `Item.json` / `ItemSourcePriority.json`, `model/role-permission-matrix.md`, state machine `itemLifecycle` (PENDING_ONBOARDING -> PENDING_APPROVAL -> ONBOARDED -> OFFBOARDED; plus SUSPENDED reversible; same shape as `supplierLifecycle`). Show-light: the list, one create/edit form, and the bulk-import action. The item lifecycle deliberately mirrors the supplier lifecycle (06).

### S07.1 Items (archetype: list)
- **Route:** `/items`
- **Roles:** Procurement/Buyer (act: create, update, import); Engineering (conditional act on engineering items, `items.create` / `items.update`); Approver (act: approve, offboard; Engineering conditional for engineering items); Administrator (view all + act).
- **Realizes:** entry point; the catalog master view.
- **Purpose:** the catalog at a glance: which items are orderable, which are pending or suspended, with the entry to add one item or bulk-import many.
- **Data shown:** `Item.code`, `Item.shortDescription` / `Item.description`, `Item.segmentId` / `Item.subSegmentId`, `Item.stockUom` / `Item.purchaseUom`, `Item.hsCode`, `Item.regulatedItem` (Badge), `Item.standardSupplierId`, `Item.status` (Badge per `itemLifecycle`, incl. SUSPENDED), `Item.isErpSynced`.
- **Primary actions:** "New Item" (-> S07.2; perm `items.create`); "Bulk Import" (-> the import action; perm `items.import`); row click -> item create/edit (S07.2).
- **Secondary actions / navigation:** faceted filter by status / segment / regulated flag; search by code / description; sort; pagination (default 10, max 50).
- **States to design:** empty ("No items yet"), loading (skeleton table), populated, error, permission-denied.
- **Shadcn components:** DataTable, Popover + Command (faceted filter), Badge (status incl. SUSPENDED, regulated, ERP-synced), Input (search), Button.
- **Acceptance criteria (believable done):** (1) `Item.status` Badges use the canonical `itemLifecycle` states and a SUSPENDED item is visibly distinguished (it blocks new ordering, reversible); (2) a `regulatedItem` = true item shows a regulated Badge; (3) "New Item" and "Bulk Import" are gated by `items.create` / `items.import`; an Engineering user sees act only on engineering items (conditional); (4) empty state shows for a fresh persona.
- **Tags:** [OE4 | RA item onboarding]

### S07.2 Create / Edit Item (archetype: form)
- **Route:** `/items/new`, `/items/:id/edit`
- **Roles:** Procurement/Buyer (act: `items.create`, `items.update`, request approval); Engineering (conditional, engineering items); Approver (act: approve, offboard).
- **Realizes:** narrative steps 3 (create item, code auto, description auto, UoMs, HS code), 4 (set source priority), 5 (request approval), 10-11 (edit revert, offboard/suspend).
- **Purpose:** capture a catalog item with its auto-generated code and description, units, HS code, regulated flag, source-priority ranking, and the lifecycle action, mirroring the supplier flow.
- **Data shown / captured:** from `Item.json` - `Item.code` (auto, DEPARTMENT/SEGMENT_CODE/SUB_SEGMENT_CODE/RUNNING_NUMBER e.g. ME/SG/SSG/000001, read-only), `Item.runningNumber` (auto, read-only), `Item.shortDescription` (entered), `Item.description` (auto-populated = segmentDesc + subSegmentDesc + shortDescription, fallback to code, read-only), `Item.segmentId` (Combobox, Category master, default SG), `Item.subSegmentId` (Combobox, default SSG), `Item.categoryId` (Combobox, reporting category), `Item.stockUom` (Select, UoM), `Item.purchaseUom` (Select, UoM), `Item.hsCode` (text, import duty classification), `Item.standardSupplierId` (Combobox over `Supplier.name`; need NOT be ONBOARDED), `Item.regulatedItem` (toggle), `Item.status` (Badge, read-only), `Item.isErpSynced` (toggle, only when ONBOARDED). SOURCE PRIORITY from `ItemSourcePriority.json` (repeatable, replace-on-update) - `priority` (integer rank, unique per item), `sourceType` (Select, ItemSourceType: MANUFACTURED | PURCHASED | SUBCONTRACTED | STOCK_TRANSFER, unique per item).
- **Primary actions:** "Save Draft" (creates/updates at PENDING_ONBOARDING); "Request Approval" (`itemLifecycle` PENDING_ONBOARDING -> PENDING_APPROVAL; guard status = PENDING_ONBOARDING; fires ITEM_APPROVAL_REQUESTED only here; audit category ADMIN); on an ONBOARDED/PENDING_APPROVAL item, saving an edit reverts to PENDING_APPROVAL and resets `isErpSynced` (axiom A10). State-aware (per `itemLifecycle`): "Approve" (Approver; PENDING_APPROVAL -> ONBOARDED; guard: code + description + stockUom + purchaseUom + >= 1 source priority), "Suspend" / "Offboard" (offboard guard ONBOARDED -> OFFBOARDED terminal; SUSPENDED reversible).
- **Secondary actions / navigation:** add/remove source-priority entry; the implicit auto-create note (see inline behaviours); link to the item list (S07.1).
- **States to design:** new (empty, status PENDING_ONBOARDING), edit (fetch, status reverts to PENDING_APPROVAL on save), validation-error (missing stock/purchase UoM, duplicate priority or sourceType within the item), saved-draft toast, approval-requested toast, permission-denied.
- **Shadcn components:** Form with Card sections, Combobox/Select (segment, sub-segment, category, UoM, standard supplier, source type), Sheet or inline rows (source-priority add/edit), Toggle (regulated, ERP sync), Input (HS code, short description), Toast.
- **Acceptance criteria (believable done):** (1) `Item.code` is generated once from the configurable DEPARTMENT/SEGMENT/SUB_SEGMENT/RUNNING_NUMBER pattern with a 6-digit zero-padded race-safe `runningNumber`, shown read-only and unique on (runningNumber, code); (2) `Item.description` is auto-populated from segment + sub-segment + short description (fallback to code) and is never empty; (3) Approve requires code + description + stockUom + purchaseUom + >= 1 source priority; (4) the source-priority set is replaced wholesale on update (not merged) and each entry is unique within the item by both `priority` and `sourceType`; (5) editing an ONBOARDED/PENDING_APPROVAL item reverts to PENDING_APPROVAL and resets `isErpSynced`; the ERP-sync toggle is available only when ONBOARDED; (6) `Item.standardSupplierId` accepts any existing supplier or null and is NOT gated on ONBOARDED (unlike the PO supplier gate).
- **Tags:** [OE4 | ISO 7.5 | RA + AB HS code]

### Inline edge behaviours shown on these screens (not separate screens)
- **Implicit auto-create from a non-UUID requisition line (e05):** an item auto-created from a free-text requisition line (02) appears here at PENDING_ONBOARDING with `description` = the free-typed string and stock/purchase UoM = the line UoM; usable on its own requisition immediately but must pass approval to be ONBOARDED for general use. Surfaced as a "created from requisition line" note on the item.
- **Bulk import all-or-nothing (e06):** the "Bulk Import" action on S07.1 opens an XLSX/CSV upload with the exact header; per-row validation accumulates every `{rowNumber, reason}`; the import is all-or-nothing (zero errors -> commit by natural key = code, duplicates last-wins, new items get an incremented runningNumber, default PENDING_ONBOARDING, audit category ADMIN only on commit); item import additionally fails the whole import if any standardSupplierCode does not resolve. Imported items still need source priority (if missing) + request-approval -> approve to reach ONBOARDED.
- **Edit reverts to re-approval (axiom A10):** saving any edit on an ONBOARDED/PENDING_APPROVAL item recomputes status to PENDING_APPROVAL, resets `isErpSynced`, and replaces the source-priority set if changed.
- **Suspend blocks new ordering (reversible):** a SUSPENDED item is blocked from new ordering but can be reinstated, unlike OFFBOARDED which is terminal (record retained for referencing requisition lines / POs).
